To calculate the **Transmission Delay** and **Propagation Delay**, we'll use the following formulas:

---

### **1. Transmission Delay**
Transmission delay is the time it takes to push all the bits of a packet onto the link. It is calculated as:

\[
\text{Transmission Delay} = \frac{\text{File Size (bits)}}{\text{Transmission Rate (bits per second)}}
\]

#### Given:
- File size = 27,000 bits
- Transmission rate = 5 Mbps = \(5 \times 10^6\) bits per second

#### Calculation:
\[
\text{Transmission Delay} = \frac{27,000}{5 \times 10^6} = 0.0054 \, \text{seconds}
\]

---

### **2. Propagation Delay**
Propagation delay is the time it takes for a bit to travel from the source to the destination. It is calculated as:

\[
\text{Propagation Delay} = \frac{\text{Distance (meters)}}{\text{Propagation Speed (meters per second)}}
\]

#### Given:
- Distance = 2,000 km = \(2,000 \times 10^3\) meters
- Propagation speed = \(2 \times 10^8\) meters per second

#### Calculation:
\[
\text{Propagation Delay} = \frac{2,000 \times 10^3}{2 \times 10^8} = 0.01 \, \text{seconds}
\]

---

### **Final Results:**
- **Transmission Delay = 0.0054 seconds**
- **Propagation Delay = 0.01 seconds**

This means:
1. A TCP connection is opened for the first request/response.
2. The connection is closed after the response is delivered.
3. A new TCP connection is opened for the next request/response, and so on.

This process is inefficient because it introduces additional overhead (e.g., TCP three-way handshake) for each request/response pair.
